# Decisions — Epic #21

## Skills-sync logic lives in the CLI package, not just a script
**File(s):** `packages/cli/src/bootstrap/skills-sync.ts`
**Date:** 2026-05-23

**Decision:** Put `diffSkills`/`syncSkills` in `@middle/cli` and have both
`scripts/sync-skills.ts` and `mm doctor` import them.
**Why:** The spec asks for two enforcement surfaces — a pre-commit hook and a
`mm doctor` drift flag. A single source of truth avoids two divergent
implementations of "are these trees byte-identical". The script is a thin CLI
wrapper; doctor calls the same function.
**Evidence:** Build spec → "Repo layout" (two-copies note + sync hook) and
sub-issue #23 acceptance ("pre-commit hook keeps them in sync", "mm doctor has a
hook point to flag drift").

## Mirror generated by the sync script, never hand-authored
**File(s):** `packages/cli/src/bootstrap-assets/skills/`
**Date:** 2026-05-23

**Decision:** Produce `bootstrap-assets/skills/` by running `sync-skills`, not by
hand-copying.
**Why:** Byte-identity is a hard invariant; generating the mirror from the
canonical guarantees it at creation time rather than relying on a careful manual
copy. The `diffSkills` test over the *shipped* paths then locks it in CI.
**Evidence:** `packages/cli/test/skills-sync.test.ts` → "the shipped canonical
and mirror are in sync".

## recommending-github-issues extracted from the spec, escapes resolved
**File(s):** `packages/skills/recommending-github-issues/SKILL.md`
**Date:** 2026-05-23

**Decision:** Extract the skill body verbatim from the build spec's embedded
code block (lines 481–686) and unescape the fence/continuation escapes
(`\`` → backtick, `\\` → `\`) rather than retyping it.
**Why:** The spec is the source of truth for this skill's text; transcribing by
hand risks drift. The escapes exist only because the text was embedded inside a
markdown code fence.
**Evidence:** `grep '```'` shows the bash fences resolved; the line-continuation
`gh issue list ... \` renders correctly.

## Pre-commit hook installed via core.hooksPath, not a committed .git/hooks file
**File(s):** `scripts/hooks/pre-commit`, `package.json` (`prepare`)
**Date:** 2026-05-23

**Decision:** Commit the hook under `scripts/hooks/` and point Git at it with
`core.hooksPath` set by the `prepare` script on `bun install`.
**Why:** `.git/hooks/` isn't tracked, so a committed hook needs an install step.
`core.hooksPath` is the lightest mechanism that needs no extra dependency (vs.
husky) and auto-installs for any contributor who runs `bun install`.
**Evidence:** Verified the hook blocks a commit when the mirror drifts and passes
when in sync.
