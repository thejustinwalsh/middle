# Issue #83: Ship creating-github-issues skill via mm init bootstrap

**Link:** https://github.com/thejustinwalsh/middle/issues/83
**Branch:** middle-issue-83

## Goal
`mm init` stamps `creating-github-issues` into target repos alongside the other two shipped skills, closing the seed → implement → recommend loop in one bootstrap. The canonical↔mirror sync model (sync-skills, pre-commit hook, `mm doctor`) covers it, `mm uninit` removes it, and the README no longer tells you to copy the skill in by hand.

## Approach
The bootstrap is **fully data-driven** — `listSkillDirs` (sync/doctor) and `listBootstrapSkills` (stage/uninit) enumerate whatever skill dirs exist; nothing hardcodes skill names. So the bulk of the work is *data*, not code:
- Promote the live `.claude/skills/creating-github-issues/SKILL.md` to canonical `packages/skills/creating-github-issues/`.
- Re-sync to generate the byte-identical mirror under `bootstrap-assets/skills/`.
- Make the four tracked copies byte-identical: canonical, mirror, and the stamped `.claude`/`.codex` copies in this repo (it's bootstrapped into itself). `.codex` currently lacks the skill; `.claude` has it.
- Extend tests to assert the third skill is staged into both CLIs, synced, and uninited.
- Relax the README "copy it in by hand" caveat.

## Phases
1. Promote canonical + re-sync mirror — create `packages/skills/creating-github-issues/`, run `bun scripts/sync-skills.ts`.
2. Self-bootstrap parity — make this repo's stamped `.claude`/`.codex` copies byte-identical to canonical (add the missing `.codex` copy).
3. Tests — extend `bootstrap-init.test.ts` (staged into both CLIs, uninited) and `skills-sync.test.ts` (no change expected — generic — but confirm the shipped-trees in-sync test now covers 3 skills).
4. README — drop the "copy it into your repo's `.claude/skills/` by hand" caveat; mention `mm init` ships it.

## Files likely to change
- `packages/skills/creating-github-issues/SKILL.md` — NEW canonical source (copied from live `.claude` copy)
- `packages/cli/src/bootstrap-assets/skills/creating-github-issues/SKILL.md` — NEW mirror (generated by sync-skills, never hand-edited)
- `.codex/skills/creating-github-issues/SKILL.md` — NEW stamped copy (self-bootstrap parity)
- `packages/cli/test/bootstrap-init.test.ts` — assert third skill staged + uninited
- `README.md` — relax the by-hand caveat in "Seeding the backlog"
